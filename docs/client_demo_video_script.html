3-Minute Video Script for CFG Enterprise Data Bot

A concise client-facing story that demonstrates the assistant as more than a chatbot: a governed commercial banking analytics assistant that understands business questions, clarifies ambiguity, generates validated SQL, creates charts, cites sources, and captures feedback.

Video Objective

Position CFG Enterprise Data Bot as an enterprise-ready analytics assistant for commercial banking data users. The story should show that it is not a generic chatbot. It is a governed assistant that can explain definitions, resolve business context, generate SQL, validate governance, visualize results, and support quality feedback.

Audience Business leaders, data owners, analytics teams, and technology stakeholders.
Duration Approximately 3 minutes.
Tone Professional, confident, practical, and business-value focused.

Key Messages to Land

Governed analytics LLM-driven reasoning Validated SQL Business definitions Clarification workflow Charts and evidence
  • The assistant helps both technical and non-technical users ask questions in plain English.
  • It answers definition questions without generating unnecessary SQL.
  • It asks clarification when business meaning is ambiguous.
  • It generates SQL only from governed metadata and validates SQL before execution.
  • It can return charts, result tables, SQL evidence, source citations, and audit details.
  • It supports feedback and quality monitoring for continuous improvement.

Condensed Voiceover Script

"Commercial banking teams depend on trusted data every day, but finding the right metric, table, definition, or SQL can take time. Business users may not know the right column names, and technical teams spend hours answering repeated data questions. CFG Enterprise Data Bot is designed to close that gap.

This is not just a chatbot. It is a governed commercial banking analytics assistant. It combines natural language, business metadata, lineage, access rules, SQL validation, charts, and feedback into one guided experience.

A user can start with a simple definition question, such as, 'What does average collected balance mean?' The assistant understands that this is informational and answers from governed business metadata without generating unnecessary SQL.

If a question is ambiguous, the assistant does not guess. For example, 'average balance' can refer to multiple certified metrics. The assistant asks a clarification question and presents governed choices.

Once the user selects the intended metric, the assistant preserves the original context, generates SQL, validates it against governed metadata, executes the query, and returns an analyst-style response with a result table and SQL evidence.

The assistant can also create charts. When a user asks, 'Plot loan utilization by month,' it selects the certified metric, groups by year month, validates the SQL, and renders a clean trend chart.

Every answer is explainable. Users can expand SQL, sources, and governance audit details to see which metrics, tables, columns, and validation rules were used.

Governance is built in. If a user asks for restricted data, such as customer names, the assistant blocks SQL generation and suggests safe alternatives.

CFG Enterprise Data Bot helps users move from question to trusted insight faster. It is conversational, but governed. It is LLM-powered, but controlled. And it gives business and technical teams a single assistant for definitions, SQL, charts, evidence, and feedback."
