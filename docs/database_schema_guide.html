Commercial Banking Database Schema Guide

A beginner-friendly guide to every generated DuckDB table, column, business definition, metric, dimension, join path, lineage record, and access policy used by the CFG Enterprise Data Bot MVP.

Database Overview

Database Filecommercial_banking.duckdb
Total Tables28
Business Tables17
Metadata Tables11

This database contains two kinds of tables: commercial banking data tables, which hold synthetic banking records, and governance metadata tables, which tell the assistant what tables, columns, metrics, dimensions, joins, lineage, and access policies are allowed.

How to Read This Guide

  • Dimension tables start with dim_ and describe business entities such as customer, account, loan, product, branch, and date.
  • Fact tables start with fact_ and contain measurable banking events or balances.
  • Metadata tables start with metadata_ and control governance, definitions, SQL safety, lineage, and search.
  • Restricted columns are blocked from generated SQL.
  • Certified metrics and dimensions are the trusted objects the bot should use for analytics.
