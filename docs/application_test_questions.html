Testing guide

20 Application Test Questions for CFG Enterprise Data Bot

A balanced set of business-user and data-engineer questions grounded in the generated commercial banking schema, metadata catalog, metrics, dimensions, lineage, and governance rules.

How to Use This Test Set

Use these questions to validate different assistant capabilities. The questions intentionally cover definition-only answers, SQL-backed analytics, charts, metadata discovery, lineage, clarification, restricted data protection, and governance evidence.

Recommended testing style: ask one question at a time in the chatbot, review the answer, expand SQL/Sources/Governance Audit where relevant, and verify whether the expected behavior is met.

Suggested Validation Checklist

  • Definition questions should not produce SQL or charts.
  • Ambiguous metrics should trigger clarification instead of guessing.
  • Follow-up clarification should preserve original context.
  • Charts should place dimensions on the X axis and metrics on the Y axis.
  • SQL should reference only governed tables and columns.
  • Restricted columns should be blocked.
  • Sources and Governance Audit should be available in collapsed sections.
  • Repeated same questions should return consistent results.
